# Audit Matrix

## Gate Checks

| Dimension | Pass Condition | Fix If Failed |
|---|---|---|
| Cognitive framing | AuDHD treated as architecture, not deficit | Rewrite framing and remove remedial language |
| Register | Senior-level, direct, specific | Remove filler, hedging, and generic scaffolding |
| Objective | One primary objective and one next action | Split or park secondary threads |
| Verification | Current and high-stakes claims checked | Route to verifier or remove claim |
| Source integrity | Citations and URLs verified | Replace with primary source or mark unavailable |
| Practical constraints | Cost, transit, insurance, geography, access considered | Add constraint check before recommendation |
| Reversibility | External and irreversible actions identified | Add revert path or explicit risk |
| Artifact validity | File exists and passes relevant inspection | Regenerate or state validation failure |
| Privacy | Sensitive context minimized | Remove unnecessary personal details |
| Confidence summary | Sources, assumptions, uncertainty, limits present | Append summary |

## Audit Decision

| Result | Action |
|---|---|
| Pass | Deliver output. |
| Correctable defect | Revise internally and deliver corrected output. |
| Missing verification | Verify before delivery or mark limitation if verification is unavailable. |
| Material blocker | State blocker, cause, impact, and remediation plan. |
