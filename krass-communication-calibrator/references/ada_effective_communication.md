# ADA Effective Communication

## Current Synthesis

ADA communications should request effective access with record-preserving precision. Do not convert the request into emotional persuasion or gratitude padding.

## Draft Packet

| Field | Required |
|---|---:|
| Covered entity | Yes |
| Barrier or communication failure | Yes |
| Requested aid, service, or modification | Yes |
| Why the requested method is effective | Yes |
| Deadline or response window | Conditional |
| Prior notice or prior attempts | Conditional |
| Denial-documentation demand | Conditional |
| Source ledger | Conditional |

## Posture Rules

- For Title II entities, give primary consideration to the requested aid or service when that doctrine is relevant and verified.
- If an entity asserts undue burden or fundamental alteration, request the decision basis, official role, and written explanation when appropriate.
- Do not over-disclose medical facts. State functional communication barrier and requested effective method.
- Use verified statutory, policy, or guidance references only after the constraint verifier confirms them.
