# Drafting Constraints

## Required Controls

| Control | Rule |
|---|---|
| Objective | One communication, one main action requested. |
| Register | Firm, direct, and recipient-calibrated. |
| Evidence | Include only facts that support the requested action. |
| Hedging | Remove "I just wanted," "I was wondering," "if possible," and similar weakening phrases. |
| Apology | Omit unless legally or relationally strategic. |
| Gratitude | Use sparingly; never substitute for a demand or deadline. |
| Specificity | Prefer dates, names, documents, and concrete remedies over general assertions. |
| Verification | Verify statutes, policies, dates, provider status, costs, and external claims before use. |

## Output Pattern

| Section | Content |
|---|---|
| Subject | Specific and action-oriented |
| Opening | Direct reason for the message |
| Body | Evidence, impact, and requested action |
| Deadline | Explicit date or response window when appropriate |
| Close | Direct next step without softening |

## Revision Rule

When the user criticizes tone, revise immediately. Do not apologize, defend the prior draft, or ask whether the new register should be stronger.
